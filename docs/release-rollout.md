# 灰度发布与回滚方案

## 灰度发布步骤

1. 在测试环境执行 `prisma migrate deploy` 并运行基础 API 冒烟测试
2. 开启小流量（10%）到新 API 实例
3. 监控指标：
   - 错误码分布（`1xxxx/2xxxx/3xxxx/5xxxx`）
   - 鉴权失败率（401/20001）
   - 关键接口时延（P95/P99）
4. 扩容到 50%，观察 30 分钟
5. 全量切换并保留旧服务只读兜底窗口

## 回滚步骤

1. 关闭新 API 路由流量，恢复旧服务路由
2. 对新版本写入进行审计（refresh token、历史与评论新增记录）
3. 如需回滚库结构，执行预置反向脚本（由 DBA 审核）
4. 复盘错误码和日志，修复后再发灰度
