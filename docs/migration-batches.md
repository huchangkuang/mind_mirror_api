# 迁移分批策略与验收标准

## 批次 1：基础框架（当前完成）

- NestJS 应用骨架
- `/api/v1` 版本策略
- 统一响应格式 `code/message/data`
- 全局异常过滤与错误码枚举
- Prisma 模型与初始迁移文件

**验收标准**
- 启动后所有 API 路径挂载在 `/api/v1`
- 成功响应统一包裹
- 异常响应统一包裹并包含错误码

## 批次 2：认证与用户（当前完成）

- 登录/注册
- access token + refresh token
- 刷新、登出（刷新令牌失效）
- 用户资料与改密

**验收标准**
- 登录返回 access/refresh token
- refresh 轮换生效，旧 refresh 失效
- 受保护接口无 token 返回未授权

## 批次 3：反馈与历史（当前完成）

- 评论列表/发布/删除/点赞
- 测试列表与历史 CRUD

**验收标准**
- 反馈接口在受保护路径下鉴权正常
- 历史写入后可按用户查询
- 删除操作不会越权影响其他用户数据

## 批次 4：题库与结果（待实施）

- MBTI 与 City Match 题库接口
- 评分与结果计算迁移

**验收标准**
- 题目和评分结果与旧项目一致
- 回归用例通过率 >= 95%
